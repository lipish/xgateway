# LLM-Link 调度与负载均衡策略详解

本文介绍 LLM-Link 网关的核心调度机制，包括负载均衡策略、健康检查、熔断保护及故障切换。

---

## 1. 核心架构概述

LLM-Link 的调度层由 `PoolManager` 统一管理，其内部整合了四个核心组件：
- **LoadBalancer (负载均衡器)**: 负责在多个健康的 **模型服务** 中选择最优节点。
- **HealthChecker (健康检查器)**: 实时监控 **模型服务** 的存活状态和延迟。
- **CircuitBreaker (熔断器)**: 实施服务自保护，防止向上游故障服务持续发送流量。
- **FailoverManager (故障切换器)**: 处理请求失败后的重试及备用节点切换逻辑。

---

## 2. 负载均衡策略 (Load Balance Strategies)

网关支持多种动态切换的策略，可通过配置灵活选择：

### 2.1 基础分配类
- **RoundRobin (轮询)**: 按顺序循环分配。最简单的公平分配方式，适用于各 **模型服务** 性能相近的场景。
- **Random (随机)**: 完全随机选择。
- **Priority (优先级)**: 始终选择 `priority` 配置最高的 **模型服务**。只有当高优先级服务不健康时，才会尝试次高优先级的服务。

### 2.2 性能导向类
- **LeastConnections (最少连接)**: 基于当前活跃连接数。优先选择负载最轻（处理中请求最少）的 **模型服务**。
- **LatencyBased (延迟导向)**: 优先选择响应最快的 **模型服务**。基于过去一段时间的平均响应时间 (P50/P99) 进行动态权重计算。

### 2.3 成本与配额类 (核心特性)
- **WeightedRoundRobin (加权轮询)**: 根据手动设置的权重比例分配流量。
- **LowestPrice (最低单价优先)**: 比较输入 Token 的单价 (`input_price`)，始终选择成本最低的 **模型服务**。
- **QuotaAware (配额感知)**: 
  - 智能筛选：自动过滤掉 `tokens_used >= quota_limit` 的 **模型服务**。
  - 降级逻辑：在配额内按单价排序；若全部配额耗尽，则回退到纯单价模式。

---

## 3. 健康检查机制 (Health Management)

`HealthChecker` 后台任务会周期性执行以下逻辑：
- **主动探测**: 定期请求 **模型服务** 的 `/v1/models` 接口（或进行 TCP 连接检查）。
- **状态流转**:
  - `Healthy`: 连续 N 次成功探测。
  - `Degraded`: 响应变慢或偶尔超时。
  - `Unhealthy`: 连续多次探测失败，服务会被暂时移出调度池。
- **指标收集**: 自动更新 **模型服务** 的平均延迟、当前 RPS 等数据，供负载均衡器参考。

---

## 4. 熔断与保护 (Circuit Breaker)

为防止请求积压和级联故障，每个 **模型服务** 都配备了独立的熔断器：
- **闭合状态 (Closed)**: 正常通行。
- **开启状态 (Open)**: 当错误率或连续失败次数超过阈值时，熔断器开启，所有发往该服务的请求将被直接拦截并上报错误。
- **半开状态 (Half-Open)**: 经过恢复等待时间（Recovery Timeout）后，允许少量请求通过。若这些请求成功，则闭合熔断器；若再次失败，则重新开启。

---

## 5. 故障切换与重试 (Failover & Retry)

当调度选中的 **模型服务** 执行失败时，`FailoverManager` 介入：
1. **智能重试**: 在满足特定条件（如网络超时、5xx 错误）时，根据“指数退避”策略进行自动原位重试。
2. **备用切换 (Fallback)**: 若重试失败，网关会查询配置的 `fallback_chain`，自动在健康的备用服务中选择下一个执行者，实现对客户端无感知的平滑切换。

---

## 6. 配置参考 (Metadata Example)

在 **模型服务** 的配置 JSON 中，可以通过以下字段控制调度行为：

```json
{
  "api_key": "sk-...",
  "base_url": "https://api.deepseek.com",
  "priority": 10,
  "weight": 80,
  "input_price": 2.0,      // 每百万 Token 输入价格 (元)
  "output_price": 8.0,     // 每百万 Token 输出价格 (元)
  "quota_limit": 1000000   // Token 总配额 (可选)
}
```
